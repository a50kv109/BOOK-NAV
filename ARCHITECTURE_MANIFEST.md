# BOOK-NAV: Архитектурный манифест (Standard V1.6)

## 1. Философия проекта
**BOOK-NAV** — это инфраструктурный стандарт для трансформации статических файловых архивов (PDF) в **машиночитаемые инженерные графы знаний**. Мы превращаем хаотичные папки с книгами в индексируемую экосистему, где каждый актив («книга») обладает собственным интеллектуальным паспортом.

## 2. Стандарт артефактов (Artifact Set)
Для каждого объекта в библиотеке (книги, ГОСТы, модули, проекты) устанавливается трехкомпонентная структура:

| Компонент | Расширение | Назначение |
| :--- | :--- | :--- |
| **SOURCE** | `.pdf` | Первоисточник (неизменяемое тело знаний) |
| **HUMAN NAVIGATION** | `*_NAV.md` | Картографический слой для человека (навигация, контекст) |
| **MACHINE PROFILE** | `*_UTILITY.yaml` | Операционный слой для ИИ (метаданные, алгоритмы, фильтры) |

## 3. Зачем это нужно?
* **Для человека:** `NAV.md` позволяет за секунды понять ценность документа, не открывая его. Это визуальная карта вашего инженерного опыта.
* **Для ИИ-агентов:** `UTILITY.yaml` — это «паспорт» актива. Агенты (Gemini, ACP, SOL) используют этот слой для мгновенной фильтрации, поиска и принятия решений по применению знания в инженерных задачах, исключая необходимость «чтения» всего текста.

## 4. Масштабируемость
Данная архитектура спроектирована для работы с библиотеками масштаба **100–1000+ объектов**.
* При 10 книгах Markdown-файлы могут быть избыточны.
* При 1000+ книгах `YAML`-профили становятся единственным способом обеспечения работоспособности автоматизированных поисковых графов и агентных систем.

## 5. Методология AAM-V1
Весь стандарт BOOK-NAV V1.6 функционирует в рамках методологии **AAM-V1 (Artsybashev’s Analysis Method)**. Это обеспечивает:
1.  **Двухэтапную проверку (Two Heroes Audit):** Мы всегда проверяем `DECLARED_ROLE` (что книга заявляет) против `ACTUAL_ROLE` (что она делает на самом деле).
2.  **Двухуровневую навигацию:** Человекочитаемый Markdown и Машиночитаемый YAML как единый контур управления.
